# 餐饮数据分析 - 使用说明

## 1. 端口说明（重要）

- **本前端仅使用端口 8080**，浏览器访问地址：**http://127.0.0.1:8080**
- **不会占用 8000 或 3000**，不影响你现有在 8000 / 3000 上运行的其他项目。

---

## 2. 运行流程

### 第一步：运行分析脚本（生成宽表、8 个 Sheet 结论表 + 前端用 report.json）

在项目目录下执行：

```bash
python main_analysis_v2_backup.py
```

运行完成后会得到：

- **宽表 Excel**（如 `餐饮业务全量宽表_YYYYMMDD_HHMMSS.xlsx`）
- **分析结论 Excel**（如 `餐饮数据化分析结论_YYYYMMDD_HHMMSS.xlsx`，含 8 个 Sheet）
- **report.json**（供前端页面读取，与上述文件同目录）

### 第二步：在浏览器中查看报告

#### 方式 A：原单页前端（根目录 index.html）

在**同一项目目录**下启动本地 HTTP 服务（仅监听 8080）：

```bash
python -m http.server 8080
```

或执行 `./启动前端服务.sh`，然后打开 **http://127.0.0.1:8080**。

#### 方式 B：新 React 仪表板（Bento 布局，符合 CURSOR_DEV_GUIDE.md）

1. 将 **report.json** 复制到 React 项目的 public 目录：`cp report.json dashboard/public/report.json`
2. 启动：`cd dashboard && npm install && npm run dev`，浏览器打开 **http://localhost:5173**
3. 或构建后：`npm run build && npm run preview`

新仪表板包含：玻璃拟态页头、销售趋势（AreaChart + MoM + 日均目标进度条）、用户画像（RFM 饼图 + 时段堆叠 + 优惠响应）、菜品表现（共现订单 + 风险预警卡片）、经营效能（员工表 + 流失统计 + 备注高频词）、全量备注洞察表（默认 5 行，可展开全部）。

若未将 report.json 放入 `dashboard/public/`，页面会提示请先运行分析脚本并放入 public 目录。

---

## 3. 小结

| 操作           | 命令 / 地址 |
|----------------|-------------|
| 生成数据与报告 | `python main_analysis_v2_backup.py` |
| 原单页前端     | `python -m http.server 8080` → http://127.0.0.1:8080 |
| 新 React 仪表板 | `cd dashboard && npm run dev` → http://localhost:5173 |
